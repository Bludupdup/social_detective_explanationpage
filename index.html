How We Built Social Detective

A step by step walkthrough of how a hackathon idea turned into a working fake account detector, from notepad code to a live machine learning powered website.

🖥️ Frontend

Building the Frontend

We started simple, writing everything directly in Notepad using plain HTML and CSS. The goal was to design an interface that felt friendly and easy to use, so we built dropdowns for selecting the platform and clean toggle buttons for yes or no questions instead of plain checkboxes. The whole layout follows a beige and red detective theme, styled like a case file, to make the experience feel like an actual investigation rather than a boring form.

🧠 Machine Learning

Training the Machine Learning Model

On the backend, we trained four different models on the same dataset: LightGBM, CatBoost, Random Forest, and Extra Trees. Each model was tested individually so we could compare how accurately it could tell fake accounts apart from real ones. We then averaged their results to get a single, more reliable overall accuracy score, since combining multiple models usually gives better and more stable predictions than relying on just one.

Once we were happy with the performance, the best performing setup was saved as a .joblib file, a format that lets us store a trained model and reload it later without retraining from scratch. This same file was first used in a separate testing script to check individual account predictions and confirm the model's real world accuracy before it ever touched the website.

⚙️ Backend

Connecting the Model to the Website

To bring everything together, we used FastAPI, a Python framework built for creating fast and simple backend servers. We added middleware to safely handle requests coming in from the frontend, then loaded our saved joblib model directly into the backend. Every input field from the website, account age, follower count, bio details, and all the other signals, is received by this Python backend, passed into the model, and turned into a live prediction that gets sent straight back to the page.
